macOS系统远程Ubuntu系统

2023-04-04
#Experiences #Unix #Windows #Ubuntu #macOS

1. VNC（Ubuntu系统需要外接显示器）

VNC的全称为 Virtual Network Computing。

第一步，Ubuntu系统先设置共享屏幕：Settings——Sharing——Remote Desktop，进行基本设置。注意⚠️：Enable Legacy VNC Protocol需要改为Require a password。

第二步，在macOS系统上打开【聚焦搜索】，即：cmd + 空格，输入 vnc://your_ip，再输入密码即可。

2. 命令行ssh访问

第一步，设置Ubuntu系统的ssh服务，见：局域网ssh远程访问Ubuntu系统。

第二步，在macOS系统上，直接在终端输入：ssh username@ip_address，命令行上传下载文件见：Ubuntu连接服务器的账号登录和上传下载文件。当然也可以用软件去进行管理，比如：Royal TSX：macOS系统的服务器管理软件。

注意⚠️：

  • 第二步，一般直接在局域网中直接可进行。
  • 第二步，如果需要在外网中操作，Ubuntu系统和macOS系统都需要安装并设置好ZeroTier（比如，Ubuntu配置ZeroTier）。
  • 第二步，Ubuntu长期开机时，需要注意防黑，见：Ubuntu防黑的经验总结。
  • 第二步，推荐使用密钥文件登陆，而不是用户名和密码的方式，见：macOS系统ssh通过密钥文件连接Ubuntu系统

3. 远程软件

远程软件，无论是内网或外网都实现macOS系统远程Ubuntu系统。

常见的软件有：

  • AnyDesk https://anydesk.com/zhs
  • TeamViewer https://www.teamviewer.cn/cn/
  • ToDesk https://www.todesk.com/

具体见：外网远程访问Ubuntu系统和远程访问工具——ToDesk。

4. Ubuntu系统不需要外接显示器

具体Ubuntu的设置见：Ubuntu虚拟显示器且远程控制（无显示器接入）。

利用AnyDesk和ToDesk软件，操作成功过。

5. 末了的话

无论使用哪种方法，都需要确保Ubuntu系统中相应的服务已安装并启动，并且在macOS系统中使用正确的IP地址或主机名进行连接。

最后，注意⚠️网络安全🔐，防止被黑。
